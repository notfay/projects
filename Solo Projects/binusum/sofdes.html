Syllabus

What you are gonna learn in this class.

Implementation

The implementation phase is critical in systems development, focusing on:

• Implementation phase
• Developing (writing programs)
• Programmers
• Analysts

Managing the Programming Process

Effective management of the programming process is essential for the successful completion of a project. Key tasks include:

• Assigning programming tasks
• Coordinating the activities
• Managing the programming schedule

Testing

Testing and documentation are often being ignoredm Testing is important because high costs associated with downtime and failures caused by software bugs

• Program is not considered finished until it has passed its testing, frequently the primary focus of the systems analysts as the system is being constructed

Test Plannning

Defines a series of tests that will be conducted, Four general stages of test:

• Unit Test
• Integration tests
• System tests
• Acceptance tests

Unit Tests

Focus on one unit, a program or a program module that performs a specific function that can be tested, Two approaches to unit testing:

• Black-box testing
• White-box testing

Integration Tests

Assess whether a set of modules or programs that must work together do so without error, Four approaches to integration testing:

• User interface testing
• Use scenario testing
• Data flow testing
• System interface testing

System Tests

Ensure that all modules and programs work together without error

• Similar to integration testing but much broader in scope
• Examine how well the system meets business requirements and its usability, security, and performance under heavy load
• Also tests the system’s documentation

Acceptance Tests

Confirm that the system is complete, meets the business need that prompted the system to be developed, and is acceptable to the users, Two stages:

• Alpha testing.
• Beta testing

Developing Documentation

Two fundamentally different types of documentation:

• System documentation
• User documentation

Types of Documentation

Three fundamentally different types of user documentation:

• Reference documents
• Procedure manuals
• Tutorials
